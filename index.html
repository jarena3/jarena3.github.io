JLS Scheduling Utility

While employed at JLS English Academy, I independently designed and built a Windows utility with the goal of reducing the beginning-of-semester workload for myself, my co-workers, and the teaching staff at other branches.
This tool automates a great deal of the repetitive work of creating and organizing classwork tracking documents, as well as student handouts. It parses .XLS files from the internal company database to build student lists, takes class information and homework schedules as input, and then constructs printable files (.DOCX, .PDF, and a few others) for use in the classroom and for distribution to co-teachers, administration staff, and students’ parents.
The utility was designed to be flexible and extensible, so that future teachers can continue to use it as a resource.

The JLS Homework Utility is a WinForms front end that stores and retrieves data as JSON. It also uses Cathal Coffey's DOCX library to write to .DOCX files, and LinqToExcel to pull student information from JLS' proprietary database.

Trixelator

Trixelator is a simple HTML5 Canvas image manipulator written in JavaScript and jQuery with Bootstrap.
Currently, Trixelator supports two primary manipulation modes: Square and Hexagonal cell divisions.
In square mode, the program divides an image into an array of cells based on the cell size slider. It then calculates the corner and center points for each cell and plots the appropriate triangles between them, based on the options the user has chosen. The triangles are then filled with the color of the pixel at the triangle's centroid (although this can be an average sample if the option is selected). At this point, if the user has chosen to do so, the triangles are overlaid with a simple noise pattern, to give some additional texture to the output image.
Hexagonal mode uses essentially the same process, but with additional points. The point coordinates are calculated differently for horizontal and vertical hexagonal arrays. The cell sizes are also scaled up slightly, to remain similar in size to square cells, for a more consistent user experience.

This project took about a week from design to completion, and as such I'm pretty proud of what it can accomplish. I do plan on working on it again in the near future, and I would like to address the following:

  • From the beginning, I planned on including a voronoi cell generation algorithm, but did not schedule enough time to implement it. This feature is the primary goal for a future expansion of the project.
  • File saving is currently done through canvas-to-DataURL, which is quick but chokes on larger files (especially .PNGs). Output should be broken into blobs and written to files, which will take longer but eliminate filesave crashing, especially on slower machines
  • The actual image manipulation processes are not well-optimize and are slower than I'd like. There is a lot of room for improvement, and should be a main goal of future development.

    Diabolic

    Diabolic is a procedural generation and weighted selection API written in C# and being developed for the Unity framework.
    Diabolic parses user-created data files and uses them to procedurally generate RPG-style items. It is designed to support all of the item behaviors and mechanics commonly found in modern RPG/ARPG games (including affixes and sockets). It is also intended to be extremely flexible and extensible. Diabolic employs the Decorator and Factory design patterns, which allows users to mix and match item functionality without needing to write subclasses, or even understand the internal mechanics of the rest of the system, and remain confident that their additions will just work with the rest of the system.
    Additionally, Diabolic employs weighted selection to select items for generation, based on user-defined weights. A user can select from a number of selection algorithms, each featuring a unique "sweetener" mechanic to skew selections based on a bonus parameter ("magic find", in the genre). These algorithms are customizable, and can be further manipulated at runtime.

    Imgruber

    Imgruber is my attempt to get my screenshot-to-forum-post pipeline as short as possible. It's a taskbar application that listens for a user-defined hotkey, at which point it opens an invisible form and allows the user to define a rectangular selection. That selection is then saved as an image and immediately uploaded to Imgur.com using the Imgur API. The URL of the uploaded image is then pushed directly to the clipboard, in the format of the user's choosing. The selectable formats follow the types that Imgur provides, allowing the user to paste a string appropriate to a forum, StackExchange question, E-mail, messenger app, etc.
    Pipeline performance using Imgruber -- taking, cropping, uploading and pasting the link to a screenshot hosted on Imgur:

    (in seconds) Attempt 1 Attempt 2 Attempt 3 Average
    Normally 74.38 66.49 82.33 74.40
    Using Imgruber 5.06 6.22 5.19 5.49

    Imgruber shortened my total pipeline by almost 1:10 — it's about 737% faster. Not bad!
    Imgur will also tweet for you (in a sense). It does not use the twitter API, but can pop up a default browser window with the imgur URL and a user-defined automatic message preloaded into a "tweet this" form.
